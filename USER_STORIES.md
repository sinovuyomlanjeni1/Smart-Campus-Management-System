# User Stories

## Overview
This document contains user stories derived from system requirements and use cases. Each user story follows the INVEST principle and includes priority and acceptance criteria.

| Story ID | User Story | Acceptance Criteria | Priority |
|----------|-----------|---------------------|----------|
| US-001 | As a student, I want to check my attendance records so that I can track my participation. | Attendance records should be updated in real time and displayed within 2 seconds. | High |
| US-002 | As a lecturer, I want to mark student attendance so that I can track class participation. | Attendance should be recorded successfully and generate a report. | High |
| US-003 | As a parent, I want to view my child's attendance report so that I can monitor their progress. | The report should be accessible via the dashboard. | Medium |
| US-004 | As a lecturer, I want to edit the class timetable so that I can manage schedule changes. | Changes should be saved and reflected in student dashboards. | High |
| US-005 | As a student, I want to receive notifications about class schedule updates so that I can stay informed. | Notifications should be sent within 5 seconds of a schedule change. | High |
| US-006 | As an admin, I want to generate reports on student attendance so that I can analyze trends. | Reports should be exportable as CSV and PDF. | Medium |
| US-007 | As an IT staff member, I want user data encrypted with AES-256 so that security compliance is maintained. | All sensitive data should be encrypted and stored securely. | High |
| US-008 | As a student, I want to book campus facilities so that I can reserve study rooms and labs. | Booking confirmation should be displayed and sent via email. | Medium |
